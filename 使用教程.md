# Dreamina Windows 使用教程

这是一套面向 Windows 用户的 Dreamina 自动化脚本包。它可以在每天晚上 `23:00` 自动扫描任务目录，调用 Dreamina CLI 提交图生视频任务，轮询状态，并把下载完成的视频整理到 `done` 目录。

本文档按两条使用路径说明：

- 图形界面：适合大多数用户
- 命令行：适合熟悉 PowerShell 的用户

## 1. 软件包包含哪些文件

当前目录中的主要文件作用如下：

- `DreaminaLauncher.ps1`：图形界面启动器，负责配置、检测环境、登录、扫描、执行、注册计划任务
- `dreamina-nightly.ps1`：主执行脚本，负责依赖检查、登录、扫描、提交、轮询、下载、状态管理
- `register-task.ps1`：注册 Windows 每日计划任务
- `config.json`：配置文件，保存输入目录、执行时间、轮询参数等
- `Start Dreamina.vbs`：日常双击启动入口，适合普通使用
- `Start Dreamina.bat`：排障启动入口，启动失败时可看到退出码

## 2. 使用前准备

使用前请确认以下条件：

- 仅支持 `Windows x64`
- 机器可以访问外网
- 允许 PowerShell 脚本运行
- 首次运行时需要联网下载或更新 `dreamina` CLI
- 首次登录 Dreamina 时需要人工完成登录授权

脚本默认会在启动时自动执行这些动作：

1. 检查本机是否已安装 `dreamina.exe`
2. 检查远端版本是否比本地新
3. 缺失时自动安装，旧版本时自动更新
4. 检查登录态，无效时触发 `dreamina login`

## 3. 任务目录怎么准备

规则固定为：一个任务 = 一个子文件夹。

每个任务文件夹必须满足：

- 目录内必须有且仅有 `1` 个 `.txt` 文件
- 目录内至少有 `1` 张图片
- 支持图片格式：`png`、`jpg`、`jpeg`、`webp`
- 提交时图片按文件名升序排列

### 单任务目录示例

```text
D:\DreaminaTasks\任务A
├─ prompt.txt
├─ 001.png
├─ 002.png
└─ 003.jpg
```

### 母目录示例

```text
D:\DreaminaTasks
├─ 项目1
│  ├─ 镜头A
│  │  ├─ prompt.txt
│  │  ├─ 001.png
│  │  └─ 002.png
│  └─ 镜头B
│     ├─ prompt.txt
│     └─ cover.jpg
└─ 项目2
   └─ 镜头C
      ├─ prompt.txt
      ├─ 01.webp
      └─ 02.webp
```

说明：

- `input_path` 可以直接指向单个任务目录
- 也可以指向母目录
- 指向母目录时，脚本只处理最深层的叶子任务目录

## 4. config.json 配置说明

默认模板如下：

```json
{
  "input_path": "D:\\DreaminaTasks",
  "poll_interval_seconds": 600,
  "max_poll_hours": 12,
  "model_version": "seedance2.0fast",
  "duration": 5,
  "ratio": "",
  "video_resolution": "",
  "login_timeout_minutes": 10,
  "task_time": "23:00",
  "log_retention_days": 30
}
```

各字段说明如下：

| 字段 | 作用 | 是否可留空 | 推荐值 / 默认值 |
| --- | --- | --- | --- |
| `input_path` | 任务输入目录，可以是单任务目录，也可以是母目录 | 否 | `D:\\DreaminaTasks` |
| `poll_interval_seconds` | 轮询任务状态的间隔秒数 | 否 | `600` |
| `max_poll_hours` | 单次运行最长轮询时长，单位小时 | 否 | `12` |
| `model_version` | Dreamina 使用的模型版本 | 否 | `seedance2.0fast` |
| `duration` | 视频时长 | 否 | `5` |
| `ratio` | 视频画幅比例 | 是 | 空字符串 `""` |
| `video_resolution` | 视频分辨率 | 是 | 空字符串 `""` |
| `login_timeout_minutes` | 登录等待超时分钟数 | 否 | `10` |
| `task_time` | 计划任务每日执行时间，格式必须是 `HH:mm` | 否 | `23:00` |
| `log_retention_days` | 日志保留天数 | 否 | `30` |

说明：

- `ratio` 和 `video_resolution` 允许留空
- `model_version` 当前可选：`seedance2.0fast`、`seedance2.0`、`seedance2.0fast_vip`、`seedance2.0_vip`
- 其他数值项都必须是正整数
- `task_time` 必须写成类似 `23:00`、`08:30` 的格式

## 6. CLI 使用步骤

如果你更习惯命令行，可以直接在当前目录打开 PowerShell 执行。

### 6.1 环境检查

```powershell
powershell -ExecutionPolicy Bypass -File .\dreamina-nightly.ps1 -CheckOnly
```

### 6.2 安装或更新 CLI

```powershell
powershell -ExecutionPolicy Bypass -File .\dreamina-nightly.ps1 -InstallOnly
```

### 6.3 登录 Dreamina

```powershell
powershell -ExecutionPolicy Bypass -File .\dreamina-nightly.ps1 -LoginOnly
```

### 6.4 扫描任务

```powershell
powershell -ExecutionPolicy Bypass -File .\dreamina-nightly.ps1 -ScanOnly
```

### 6.5 立即执行一次

```powershell
powershell -ExecutionPolicy Bypass -File .\dreamina-nightly.ps1 -ConfigPath .\config.json -RunOnce
```

### 6.6 用命令覆盖输入目录

如果你不想改 `config.json`，可以临时指定输入目录：

```powershell
powershell -ExecutionPolicy Bypass -File .\dreamina-nightly.ps1 -ConfigPath .\config.json -InputPath "D:\DreaminaTasks" -RunOnce
```

### 6.7 注册每日计划任务

```powershell
powershell -ExecutionPolicy Bypass -File .\register-task.ps1 -ConfigPath .\config.json
```

### 6.8 查看 Dreamina 计划任务

```powershell
powershell -ExecutionPolicy Bypass -File .\register-task.ps1 -List
```

### 6.9 取消指定计划任务

```powershell
powershell -ExecutionPolicy Bypass -File .\register-task.ps1 -Unregister -TaskName DreaminaNightly
```

### 6.10 取消全部 Dreamina 计划任务

```powershell
powershell -ExecutionPolicy Bypass -File .\register-task.ps1 -UnregisterAll
```

说明：

- `-ConfigPath`：指定配置文件路径，不传时默认读取当前目录的 `config.json`
- `-InputPath`：临时覆盖配置里的 `input_path`
- `-RunOnce`：立刻执行完整流程
- `-CheckOnly`：仅检查环境
- `-InstallOnly`：仅安装或更新 CLI
- `-LoginOnly`：仅登录
- `-ScanOnly`：仅扫描目录并输出可读 JSON
- `-List`：列出 Dreamina 计划任务
- `-Unregister`：取消 `-TaskName` 指定的计划任务
- `-UnregisterAll`：取消所有名称以 `Dreamina` 开头的计划任务

## 8. 任务状态与重试规则

脚本会把任务状态写入 `state/tasks.json`，主键是任务目录绝对路径。
>>>>>>> f1347f57a20eb222065b4721acfb8415734f5c59

核心规则如下：

- 成功任务会永久跳过
- 失败任务只有在 `txt` 或图片内容发生变化后，才允许重新提交
- 已提交但未完成的任务，下次运行会优先续查，不会重复提交
- 提交失败时会自动重试一次
- 任务终态失败时也只会自动重提一次
- 第二次仍失败，就标记为最终失败
- 默认每 `600` 秒轮询一次
- 默认最长轮询 `12` 小时

## 9. 常见问题 / 排障

### 9.1 双击后闪退怎么办

先双击 `Start Dreamina.bat`，不要先用 `Start Dreamina.vbs`。  
`bat` 会保留退出信息，便于定位问题。

### 9.2 为什么第一次运行会要求登录

因为脚本会先检查 `dreamina user_credit`。如果登录态不存在或已失效，就会自动触发 `dreamina login`。这是正常行为。

### 9.3 为什么扫描不到任务

请检查：

- 目录里是否只有 `1` 个 `.txt`
- 是否至少有 `1` 张支持格式的图片
- 图片扩展名是否为 `png/jpg/jpeg/webp`
- `input_path` 是否指到了正确目录
- 任务目录是否在更深层子目录里

### 9.4 为什么同一个任务不会重复提交

因为脚本会记录任务目录的内容指纹。成功任务会永久跳过；失败任务只有在 prompt 或图片内容变化后才会重新提交。

### 9.5 为什么打不开 done 或 logs

常见原因有：

- 还没有真正执行过任务
- 当前 `input_path` 不存在
- 当前填的是单任务目录，输出目录实际在它的母目录下

### 9.6 更新失败会不会影响继续使用

如果本地已经有可用的旧版 `dreamina.exe`，更新失败时脚本会记录警告并继续主流程。  
只有“本地没有 CLI 且下载失败”时，脚本才会直接退出。

### 9.7 计划任务为什么要求用户登录 Windows

因为登录 Dreamina 可能需要人工交互，所以计划任务会使用当前用户上下文，并要求在用户已登录 Windows 会话时运行。

## 10. 推荐使用流程

如果你是第一次使用，建议按下面流程操作：

1. 准备好任务目录
2. 双击 `Start Dreamina.vbs`
3. 点击 `检测环境`
4. 点击 `安装/更新 CLI`
5. 点击 `登录`
6. 选择输入目录并保存配置
7. 点击 `扫描任务`
8. 点击 `立即执行`
9. 确认 `done` 中能看到视频
10. 最后点击 `注册计划任务`
11. 如果后续不想自动执行，点击 `取消选中` 或 `取消全部`
